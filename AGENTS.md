# AGENTS.md — HMS Helping Hand

This file documents the conventions, constraints, and architecture decisions for AI agents (Lovable, Codex) and human contributors working on this project.

## Project

**Helping Hand HMS** — single-tenant enterprise Elderly Home Management System for 伸手助人協會 (a Hong Kong elderly care NGO).

- Bilingual: Traditional Chinese (`zh-HK`, default) + English (`en`)
- Sprint cadence: S1-A (foundation, Lovable), S1-B (backend wiring, Codex), S2+ (modules)

| Layer | Choice |
|---|---|
| Runtime | Cloudflare Worker via TanStack Start v1 |
| Framework | React 19 + TypeScript (strict) + Vite 7 |
| Routing | TanStack Router file-based (`src/routes/`) |

## Design system rules (HARD)

1. **NEVER hardcode hex colors** in component code. Always use CSS custom properties from `src/styles.css` (e.g. `var(--bg-surface)`, `var(--text-primary)`).
2. **NEVER hardcode user-facing strings** in JSX. Always use `t("namespace.key")` and add the key to BOTH `zh-HK.json` and `en.json`.
3. **NEVER add a max-width container** on operational content (Admin/WardTablet shells). Content fills the full available width with shell gutter padding only.
4. **Border radius** uses `var(--radius-*)` tokens — never arbitrary `px`.
5. **Spacing** uses `var(--space-*)` tokens or Tailwind utilities backed by them.
6. **Status semantics** (alert/task/emar/icp/bed status, incident severity) live in i18n under their dedicated namespaces.

## Authentication

- Real auth lives in `src/lib/AuthContext.tsx`. Always consume via `useAuth()`.
- Protected routes wrap content in `<ProtectedRoute>` from `src/lib/ProtectedRoute.tsx`.
- The mock `src/lib/auth.ts` from S1-A has been removed.
- **Auth state listener pattern**: subscribe to `onAuthStateChange` BEFORE calling `getSession()` to avoid losing the initial `SIGNED_IN` event.

---

## Roles & RLS (apply when DB tables are added)

- Roles MUST live in a separate `user_roles` table — NEVER on `profiles` or `auth.users`. Doing otherwise creates a privilege-escalation vector.
- Define an `app_role` enum and a `SECURITY DEFINER` function `public.has_role(_user_id uuid, _role app_role)` to break recursive RLS chains.
- All RLS policies referencing role checks must call `public.has_role(auth.uid(), 'admin')` (etc.) — never inline `EXISTS (SELECT ... FROM user_roles)`.

## Server runtime constraints

The SSR entry runs in a Cloudflare Worker with `nodejs_compat`. Do NOT use:

- `child_process` (spawn/exec) — non-functional stub
- `sharp`, `canvas`, `puppeteer` — native binaries
- `fs.watch`, `os.cpus()`, `os.networkInterfaces()`
- Any package requiring a real OS filesystem or arbitrary host paths

If a workload requires Node-only deps, push it to an external HTTP service or a separate edge function.
